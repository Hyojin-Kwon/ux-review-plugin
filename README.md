# UX Review with AI — Figma Plugin

피그마에서 프레임을 선택하고, Claude를 통해 UX 검증을 실행하는 플러그인입니다.

## 파일 구조

```
ux-review-plugin/
├── manifest.json    # 피그마 플러그인 설정
├── code.js          # 메인 스레드 (프레임 export, 메시지 통신)
├── ui.html          # UI (검증 설정, API 호출, 결과 표시)
└── README.md
```

## ⚠️ 사용 시 주의사항

### 분석 결과의 한계
- 이 플러그인은 **UX 일반론 기반**으로 화면을 분석합니다
- LINE의 고유한 디자인 문법, 기획 스펙, 브랜드 가이드라인은 반영되지 않습니다
- 시장별(JP/TH/TW) 문화적 맥락이나 사용자 행동 패턴은 제한적으로만 고려됩니다

### 권장 사용법
1. **컨텍스트 보강 필수**: 분석 전 프롬프트에 다음 정보를 추가하세요
   - LINE Design System(LDS) 가이드라인
   - 해당 서비스의 기획 스펙 문서
   - 시장별 특수사항 (예: 일본 시장 문자 밀도 선호도)
   
2. **결과 해석 시 주의**
   - AI 제안사항은 "참고용 인사이트"로 활용
   - 실제 적용 전 디자이너/기획자 검토 필수
   - LINE 서비스 전체 맥락에서 타당성 재검증

3. **최적 활용 시나리오**
   - 초기 아이디어 검증 단계
   - 크로스마켓 비교 시 방향성 탐색
   - 디자인 리뷰 전 사전 체크리스트

> 💡 **Tip**: "이 화면은 LINE [서비스명]의 [기능]입니다. LDS 원칙상 [특정 규칙]을 따라야 합니다" 같은 프롬프트를 먼저 입력하면 더 정확한 분석을 받을 수 있습니다.

## 설치 방법

1. Figma Desktop → Plugins → Development → Import plugin from manifest
2. `manifest.json` 파일을 선택
3. 플러그인 실행

## 초기 설정

### API 설정
1. **설정** 탭에서 API 방식 선택
   - **Anthropic API**: API Key 입력 (`sk-ant-...`)
   - **AWS Bedrock**: 사내 프록시 서버 필요 (code.js에 엔드포인트 추가)
2. 모델 선택 (기본: Claude Sonnet 4)

### 컨텍스트 설정
1. **컨텍스트** 탭 이동
2. UX 원칙, 유저 시나리오, 디자인 제약 조건을 마크다운으로 작성
3. 기본 템플릿이 제공되며, 프로젝트에 맞게 수정 가능
4. **컨텍스트 저장** 클릭 (로컬 스토리지에 저장됨)

## 사용 방법

### 화면 단위 UX 리뷰
1. 피그마에서 프레임 1개 선택
2. 검증 유형: "화면 단위 UX 리뷰" 선택
3. 타겟 시장 선택 (복수 선택 가능)
4. **UX 검증 실행** 클릭

### 플로우 검증
1. 피그마에서 프레임 여러 개를 순서대로 선택
2. 검증 유형: "플로우 검증" 선택
3. **UX 검증 실행** 클릭
4. 선택한 프레임 순서대로 플로우가 분석됩니다

### 시장별 비교 검증
1. 동일 화면의 시장별 버전 프레임 선택
2. 검증 유형: "시장별 비교 검증" 선택
3. 비교 대상 시장 칩 선택
4. **UX 검증 실행** 클릭

## 커스터마이즈

### 컨텍스트 수정
컨텍스트 탭에서 아래 항목을 프로젝트에 맞게 수정하세요:
- UX 원칙: 팀의 핵심 UX 기준
- 유저 시나리오: 검증 대상 화면별 유저 맥락
- 시장별 고려사항: 로컬라이제이션 기준

### Bedrock 연동
사내 AWS Bedrock을 사용하는 경우:
1. Bedrock 프록시 서버 구축 (CORS 허용)
2. `ui.html`의 `callClaudeAPI()` 함수에서 Bedrock 프록시 URL 설정
3. `manifest.json`의 `networkAccess.allowedDomains`에 프록시 도메인 추가

## 제한사항

- 프로토타입 버전으로, Anthropic Direct API 모드만 즉시 동작합니다
- Bedrock 모드는 사내 프록시 서버 세팅이 필요합니다
- 이미지는 2x 해상도 PNG로 export됩니다
- 컨텍스트와 설정은 브라우저 로컬 스토리지에 저장됩니다
