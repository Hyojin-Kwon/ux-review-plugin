# UX Review Context Package — LINE App

> 이 문서는 AI 기반 UX 검증 시 컨텍스트로 함께 제공하는 기준 문서입니다.
> 스크린샷과 함께 이 문서를 첨부하고, 프롬프트 템플릿에 따라 검증을 요청하세요.

---

## 1. UX 원칙

LINE 앱 디자인 검증 시 아래 기준을 우선적으로 적용합니다.

<!-- ✏️ 수정 가능: 팀/프로젝트 상황에 맞게 원칙을 추가하거나 우선순위를 조정하세요. -->

### 핵심 원칙

- **정보 위계**: 화면 내 가장 중요한 정보가 시각적으로 가장 먼저 인지되어야 한다.
- **태스크 효율**: 유저의 핵심 태스크는 최소 단계(이상적으로 1-2탭)로 완료 가능해야 한다.
- **인지 부하 최소화**: 한 화면에서 유저가 판단해야 할 선택지는 3-5개 이내로 제한한다.
- **일관성**: LDS(LINE Design System) 컴포넌트와 패턴을 따르며, 화면 간 동일 기능은 동일 패턴으로 표현한다.
- **피드백 명확성**: 유저 액션에 대한 시스템 반응(로딩, 성공, 에러)이 즉각적이고 명확해야 한다.

### 접근성 기준

- 텍스트 최소 크기: 12sp (본문 기준)
- 터치 타겟 최소 크기: 44x44pt
- 색상 대비: WCAG AA 이상 (4.5:1)
- 아이콘은 단독 사용 시 반드시 레이블 동반

## 2. 유저 시나리오

검증 대상 화면별 유저 맥락을 아래에 정리합니다.

<!-- ✏️ 수정 가능: 스프린트 검증 대상 화면에 맞게 자유롭게 추가/삭제/수정하세요.
     각 화면마다 유저, 진입 맥락, 기대 행동, 검증 포인트를 2-3줄로 정리하면 충분합니다.
     아래는 작성 예시입니다. 실제 검증 대상에 맞게 교체해주세요. -->

### 홈 탭

- **유저**: 기존 사용자 (일 1회 이상 접속)
- **진입 맥락**: 앱 실행 직후 도달하는 첫 화면. 읽지 않은 메시지 확인이 1차 목적.
- **기대 행동**: 미읽 메시지가 있는 채팅방을 즉시 식별 → 탭하여 진입
- **검증 포인트**: 미읽 표시(뱃지)의 가시성, 최근 대화 정렬 순서의 직관성

### 채팅방

- **유저**: 1:1 또는 그룹 채팅 참여자
- **진입 맥락**: 홈 탭 또는 푸시 알림을 통해 진입. 대화 이어가기 또는 새 메시지 확인 목적.
- **기대 행동**: 최신 메시지 확인 → 입력창에 타이핑 → 전송
- **검증 포인트**: 입력창 접근성, 미디어/스티커 전송 버튼 발견 용이성, 스크롤 시 시간 구분 명확성

### 지갑 탭 (LINE Pay)

- **유저**: 결제/송금 목적 사용자
- **진입 맥락**: 하단 탭 바에서 지갑 탭으로 이동. 잔액 확인 또는 결제 수단 접근 목적.
- **기대 행동**: 잔액 즉시 확인 → QR결제 또는 송금 기능에 1탭으로 접근
- **검증 포인트**: 잔액 정보의 가시성, 핵심 액션(결제/송금) CTA의 우선순위, 보안 정보 노출 적절성

### 뉴스/디스커버 탭

- **유저**: 콘텐츠 소비 목적 사용자
- **진입 맥락**: 하단 탭 바에서 이동. 시간 소비(browsing) 목적이 강함.
- **기대 행동**: 관심 콘텐츠 탐색 → 탭하여 상세 진입
- **검증 포인트**: 콘텐츠 카드 간 구분 명확성, 광고와 오가닉 콘텐츠 구분 가능 여부, 스크롤 피로도

## 3. 디자인 제약 조건

<!-- ✏️ 수정 가능: 프로젝트에 해당하는 제약 조건을 추가하세요.
     예) 사용 가능한 컴포넌트 목록, 토큰 규칙, 플랫폼별 제약 등 -->

### 시장별 고려사항

- **일본 (JP)**: 정보 밀도에 대한 허용도가 높음. 텍스트 중심 UI 선호. 가격/혜택 정보는 상세하게 노출해도 무방.
- **태국 (TH)**: 비주얼/컬러풀한 표현 선호. 프로모션 배너의 시각적 임팩트 중요. 텍스트는 간결하게.
- **대만 (TW)**: JP와 유사하나 Traditional Chinese 특성상 텍스트 영역 여유 필요. 라인 하이트 주의.

## 4. 프롬프트 템플릿

아래 템플릿을 복사하여 사용하세요. `[ ]` 안의 내용을 상황에 맞게 채워 넣으면 됩니다.

<!-- ✏️ 수정 가능: 검증 관점이나 질문 항목을 프로젝트에 맞게 추가/수정하세요.
     자주 묻는 질문이 생기면 새로운 템플릿으로 추가해두면 반복 사용할 수 있습니다. -->

### Template A: 화면 단위 UX 리뷰

```
위에 첨부한 문서는 UX 검증 기준입니다. 아래 스크린샷을 이 기준에 따라 검증해주세요.

- 검증 대상 화면: [화면 이름]
- 타겟 시장: [JP / TH / TW]

아래 관점에서 분석해주세요:
1. 정보 위계: 가장 중요한 정보가 시각적으로 우선 인지되는가?
2. 태스크 효율: 유저 시나리오상 핵심 태스크까지 몇 단계가 필요한가?
3. 인지 부하: 한 화면에서 유저가 처리해야 할 정보량이 적절한가?
4. 시장 적합성: 해당 시장 사용자 특성에 맞는가?

각 항목별로 문제 지점과 개선 제안을 구체적으로 제시해주세요.
```

### Template B: 플로우 검증

```
위에 첨부한 문서는 UX 검증 기준입니다. 아래 스크린샷들은 하나의 유저 플로우를 구성하는 연속 화면입니다.

- 플로우 이름: [플로우 이름]
- 플로우 설명: [유저가 A에서 시작해서 B를 완료하는 과정]
- 타겟 시장: [JP / TH / TW]

아래 관점에서 분석해주세요:
1. 단계 수: 이 플로우는 최적의 단계 수인가? 줄일 수 있는 단계가 있는가?
2. 화면 간 연결성: 각 화면에서 다음 화면으로의 전환이 자연스러운가?
3. 이탈 지점: 유저가 플로우를 중단하거나 혼란을 느낄 수 있는 지점은?
4. 에러 복구: 실수 시 이전 단계로 돌아가는 것이 용이한가?

각 단계별로 분석하고 전체 플로우 개선 제안을 제시해주세요.
```

### Template C: 시장별 비교 검증

```
위에 첨부한 문서는 UX 검증 기준입니다. 아래 스크린샷들은 동일 화면의 시장별 버전입니다.

- 화면 이름: [화면 이름]
- 비교 시장: [JP / TH / TW 중 해당하는 것]

각 시장 버전을 해당 시장의 사용자 특성 기준으로 비교 검증해주세요:
1. 레이아웃: 정보 밀도가 해당 시장 사용자 기대치에 맞는가?
2. 텍스트: 언어별 텍스트 길이 차이가 레이아웃에 영향을 주는가?
3. CTA: 핵심 액션의 배치와 강조가 시장별로 적절한가?
4. 비주얼: 컬러/이미지 사용이 시장 선호도에 부합하는가?
```
