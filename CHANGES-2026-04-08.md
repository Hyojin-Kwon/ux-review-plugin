## 1. OpenAI 내장 프록시 (사용자 설정 불필요)

### 구조
Figma iframe → Cloudflare Worker (API Key 내장) → OpenAI API
사용자는 OpenAI 선택만 하면 바로 사용 가능. API Key/Proxy URL 입력 불필요.

**프록시 URL (하드코딩):** `https://holy-lake-737b.bymyhyo.workers.dev`
**API Key:** Cloudflare Worker 환경변수 `OPENAI_API_KEY`에 저장 (Secret, 암호화)

### 변경 파일

#### manifest.json
`allowedDomains`에 추가:
```json
"https://*.workers.dev",
"https://holy-lake-737b.bymyhyo.workers.dev"
```

#### ui.html — callOAICompat 함수 위에 상수 추가
```js
const OPENAI_PROXY = "https://holy-lake-737b.bymyhyo.workers.dev";
```

#### ui.html — callOAICompat 함수
OpenAI일 때 프록시 URL로 호출, Authorization 헤더 없음 (Worker가 처리):
```js
if (apiMode==="openai") {
  endpoint = OPENAI_PROXY;
  headers = { "Content-Type":"application/json" };
}
```

#### ui.html — 설정 UI (OpenAI 섹션)
API Key / Proxy URL 입력 필드 제거, 안내 메시지로 교체:
```html
<div style="font-size:11px;color:#888;line-height:1.5;padding:8px 0">사내 API가 내장되어 있어 별도 설정 없이 바로 사용 가능합니다.</div>
<input type="hidden" id="openai-api-key" />
<input type="hidden" id="openai-proxy-url" />
```

#### ui.html — 상태 체크
OpenAI 선택 시 항상 OK:
```js
else if (m==="openai") { ok=true; label=t("status_openai_ok"); }
```

#### openai-proxy-worker.js (새 파일)
Cloudflare Worker 코드. `env.OPENAI_API_KEY`에서 키를 읽어 OpenAI로 프록시.
Worker 배포 완료, 환경변수 설정 완료.

## 3. 진행률 표시

### CSS 추가
```css

.loading-progress-bar { height: 3px; background: #444; border-radius: 2px; overflow: hidden; }
.loading-progress-fill { height: 100%; background: #06C755; border-radius: 2px; width: 0%; transition: width 0.6s ease; }
.loading-step { font-size: 10px; color: #666; margin-top: 6px; transition: opacity 0.3s; }
```

### HTML
3개 메뉴 로딩 영역에 각각 추가:
```html
<div class="loading-progress">
  <div class="loading-progress-bar"><div class="loading-progress-fill" id="{prefix}-progress-fill"></div></div>
  <div class="loading-step" id="{prefix}-step"></div>
</div>
```
prefix: `loading` (UX Review), `ab-loading` (A/B), `uc-loading` (UC)

### JS
`startProgress(prefix)` / `stopProgress(prefix)` 함수 추가.
5단계 메시지: 스크린샷 준비 → 화면 구조 분석 → UX 패턴 검토 → 인사이트 정리 → 리포트 생성.
showLoading / showABLoading / UC 로딩 표시/숨김 시 호출.
